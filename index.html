Data Types

Primitive Types

These are the most basic data types. They store simple values directly in memory.

  • Numeric: byte, short, int, long (integers); float, double (floating-point numbers)
  • Character: char (a single character)
  • Boolean: boolean (true or false)
int age = 25;
double price = 19.99;
char initial = 'J';
boolean isLoggedIn = true;

Reference Types

These variables store a memory address (a "reference") to where the actual object is located. They do not store the object itself.

  • Objects: Instances of classes (e.g., String, Scanner, or any custom class).
  • Arrays: Collections of elements of the same type.
// str holds a reference to a String object
String str = new String("Hello, World!");

// numbers holds a reference to an array object
int[] numbers = {1, 2, 3};

Control Structures & Arrays

Looping Statements

Loops execute a block of code repeatedly as long as a condition is true.

  • for loop: Used when the number of iterations is known.
  • while loop: Checks the condition first, then executes. Good for an unknown number of iterations.
  • do-while loop: Executes at least once, then checks the condition.
  • Enhanced for-loop (for-each): A simpler way to iterate over arrays or collections.

Arrays

Arrays are fixed-size containers for elements of the same type.

// 1D Array
int[] oneD = {10, 20, 30};

// 2D Array (Matrix)
int[][] twoD = { {1, 2}, {3, 4} };

// 3D Array (Cube)
int[][][] threeD = { { {1}, {2} }, { {3}, {4} } };

// Jagged Array (rows have different numbers of columns)
int[][] jagged = new int[3][];
jagged[0] = new int[3];
jagged[1] = new int[5];
jagged[2] = new int[2];

OOP Fundamentals

Object-Oriented Programming is a paradigm based on the concept of "objects". The four main pillars are:

1. Encapsulation

Bundling data (fields) and the methods that operate on that data into a single unit (a class). Access to the data is often restricted using `private` modifiers, and public methods (getters/setters) are provided to control access.

2. Inheritance

A mechanism where a new class (subclass) acquires the properties and behaviors of an existing class (superclass). This promotes code reuse.

3. Polymorphism

The ability of an object to take on many forms. The most common use is when a parent class reference is used to refer to a child class object. This allows for method overriding, where a child class can provide its own implementation of a parent's method.

4. Abstraction

Hiding complex implementation details and showing only the essential features of the object. This is achieved using abstract classes and interfaces.

Overloading

Constructor Overloading

Providing multiple constructors for a class with different parameter lists, allowing objects to be created in various ways.

Constructor Chaining: Use this() to call another constructor from within a constructor. It must be the first statement.
public class Box {
    // Constructor for a cube. It calls the more general constructor.
    public Box(double side) {
        this(side, side, side); // Calls the 3-argument constructor below.
    }

    // Constructor for a custom box.
    public Box(double w, double h, double d) { /* ... */ }
}

Method Overloading

Defining multiple methods with the same name but different parameters. The correct method to call is determined by the arguments passed.

class Calculator {
    int add(int a, int b) {
        return a + b;
    }
    double add(double a, double b) {
        return a + b;
    }
}

Inheritance

Inheritance allows a child class (subclass) to acquire the properties and behaviors of a parent class (superclass).

Method Overriding: A subclass can provide a specific implementation for a method that is already defined in its parent class. Use the @Override annotation for safety.
class Animal {
    public void makeSound() { System.out.println("Generic sound"); }
}

class Dog extends Animal {
    @Override
    public void makeSound() { System.out.println("Woof!"); }
}

Types of Inheritance

  • Single Inheritance: A class inherits from one superclass. (A -> B)
  • Multilevel Inheritance: A class inherits from a subclass, forming a chain. (A -> B -> C)
  • Hierarchical Inheritance: Multiple classes inherit from a single superclass. (B <- A -> C)
  • Java does not support Multiple Inheritance (A, B -> C) with classes to avoid the "Diamond Problem." This is solved using interfaces.

Keywords

this and super

  • this: A reference to the current object instance. Used to disambiguate instance variables from local variables or to call another constructor in the same class (this()).
  • super: A reference to the immediate parent class object. Used to access parent class members or to call the parent class constructor (super()).
class Dog extends Animal {
    String breed;
    public Dog(String name, String breed) {
        super(name); // MUST be 1st line: calls parent constructor.
        this.breed = breed; // Disambiguates field from parameter.
    }
    public void display() {
        super.makeSound(); // Calls parent's makeSound() method.
    }
}

static and final

  • static: The member belongs to the class itself, not an object. Shared by all instances.
  • final: Makes an entity unchangeable (a constant variable, a non-overridable method, or a non-inheritable class).
final class Utility { // This class cannot be extended.
    // A constant shared by all parts of the program.
    public static final double PI = 3.14159;

    // A static method can be called without creating an object.
    public static int add(int a, int b) { return a + b; }
}

Abstract Classes

An abstract class is a restricted class that cannot be used to create objects. It serves as a blueprint. An abstract method is a method declared without an implementation, forcing subclasses to provide it.

Core Purpose: To achieve abstraction by defining a contract that all concrete subclasses must adhere to, ensuring they implement essential behaviors.
abstract class Shape { // Cannot be instantiated.
    // Abstract method - no body. Subclasses MUST implement this.
    public abstract double calculateArea();

    // Concrete method - has a body and is inherited normally.
    public void display() { System.out.println("I am a shape."); }
}

class Circle extends Shape {
    @Override
    public double calculateArea() { return Math.PI * 5 * 5; }
}

Sample Program

This simple program demonstrates multiple concepts working together: an abstract class, inheritance, constructor overloading, and the `this`, `super`, and `final` keywords.

// 1. Abstract Class: A blueprint for a person.
abstract class Person {
    String name;
    public Person(String name) {
        this.name = name; // 'this' refers to the current object's field.
    }
    // Abstract method: Subclasses must provide an implementation.
    public abstract String getDetails();
}

// 2. Concrete Class with Inheritance
class Student extends Person {
    int studentId;

    // 3. Constructor Overloading
    public Student(String name) {
        // Calls the other constructor in this class.
        this(name, -1); // -1 for unknown ID.
    }

    public Student(String name, int studentId) {
        super(name); // Calls the parent (Person) constructor.
        this.studentId = studentId;
    }

    @Override
    public String getDetails() {
        return "Name: " + name + ", ID: " + studentId;
    }

    // 4. Final method: Cannot be overridden by subclasses of Student.
    public final void printSchool() {
        System.out.println("Attends University of Java");
    }
}

// Main class to run the program
public class SchoolDemo {
    public static void main(String[] args) {
        Student student1 = new Student("Alice", 12345);
        Student student2 = new Student("Bob"); // Uses overloaded constructor

        System.out.println(student1.getDetails());
        student1.printSchool();
        
        System.out.println(student2.getDetails());
    }
}
